Atelier Pratique Symfony n°1

Cette atelier à pour objectif de vous permettre de pratiquer les notions vu en formation.

Cadre & Explication

Félicitation, vous venez d’intégrer notre équipe de développement PHP. Notre entreprise travail au développement d’une solution de partage de bien entre particulier. PartageToo

Consigne

Nous comptons sur vous afin de démarrer le développement de notre application. Une réunion est prévue en fin de journée avec notre comité afin de leur présenter notre avancement.

Voici les éléments dont nous aurions besoin en priorité :

1. Installation de Symfony :

Procédez à l’installation de la dernière version stable de Symfony. Il vous ai demander également de déployer cette installation sur GitHub en Public.

Vous enverrez le lien de votre GitHub (qui servira de correction) à notre Tech Lead.

2. Base de données

Nous vous demandons de mettre en place la base de donnée de notre application avec Symfony & Doctrine. Vous devez créer les entités en vous appuyant sur les éléments fournis.

Eléments de conceptions :

Pour le moment, nous souhaitons simplement que n’importe qu’elle visiteur de notre site internet puisse déposer une annonce afin de mettre en partage son bien sur notre plateforme. Aucune inscription, ni connexion n’est requise.

Généralement, une annonce se compose d’un titre et d’une description.

3. Fixtures

Une fois votre entité mise en place, vous procéderez a la création de Fixtures. Nous avons besoin de données de démonstration pour la présenter de notre avancement.

BONUS : Vous utiliserez la lbrairie de Faker pour générer des données aléatoire.

4. Page d’Accueil

Pour notre réunion de comité prévue en fin de journée, nous vous demandons de mettre en place une seule page et une seule route. Celle de la page d’accueil.

Pour le moment nous souhaitons afficher sur la page d’accueil, la totalité des annonces de notre base de donnée par ordre décroissant. C’est à dire les annonces les plus récentes en premier.

Nous n’attendons pas de graphisme particulier, cependant notre tech lead recommande l’utilisation de bootstrap et une version responsive sera mieux perçu.

Remise des éléments

A la fin de votre réalisation, vous ferez une commit de votre travail et un push sur votre branche master ou main.
